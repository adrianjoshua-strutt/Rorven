# Definition of Ready

A feature is ready for implementation when:

- the problem and user value are clear,
- acceptance criteria are testable,
- non-goals are explicit,
- affected modules and ports are identified,
- architecture impact is reviewed,
- a new ADR is created when required,
- security and secret implications are assessed,
- persistence and migration impact is known,
- supported upgrade origins and the canonical target model are identified,
- any proposed compatibility layer explains why migration is insufficient and has a draft exception ADR,
- recovery and idempotency behavior is defined,
- observability requirements are defined,
- rollout and rollback are understood,
- the test plan exists.
