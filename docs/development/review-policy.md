# Review policy

## Required review perspectives

- domain and module-boundary review
- security and permissions review
- persistence and recovery review
- adapter-contract review
- test evidence review
- operational impact review

## Reviewer independence

Where practical, the reviewing agent should not be the same agent that produced the implementation.

## Prohibited approval basis

- model consensus without tests
- passing happy-path tests only
- undocumented provider-specific workaround
- unreviewed permission broadening
- unverified recovery claims
